## Tab Content Specifications

### Tab 1: 概览 (Overview)

**核心区域：TOP3 精华论点（可展开卡片）**

概览页不再显示评分统计卡（阅读难度/内容深度等），而是聚焦于书中最重要的 3 个核心论点，以可展开的交互卡片呈现。每个卡片包含：

- **折叠态**：排名徽章、评分、标题、一句话摘要、展开按钮
- **展开态**：4 个子板块
  - 📌 核心主张（完整论述）
  - 🔗 推理链（可视化节点链）
  - 💡 实践案例（2-3 个具体例子，含正例与反例对比）
  - ⚡ 一句话行动指南（高亮渐变背景）

```html
<section id="tab-overview" class="tab-panel active">
  <div class="section-title">
    <h2>🏆 TOP3 精华论点</h2>
    <p>点击卡片展开详细解读与案例</p>
  </div>
  <div class="top3-list">
    <!-- TOP 1 -->
    <div class="top3-card" onclick="toggleTop3(this)">
      <div class="top3-header">
        <span class="top3-rank">🥇 TOP 1</span>
        <span class="top3-score">{X}/10</span>
      </div>
      <div class="top3-title">{论点标题}</div>
      <div class="top3-brief">{一句话摘要}</div>
      <div class="top3-arrow">▼ 展开详情</div>
      <div class="top3-detail">
        <div class="top3-block">
          <h4>📌 核心主张</h4>
          <p>{完整主张描述，含作者亲身经历验证}</p>
        </div>
        <div class="top3-block">
          <h4>🔗 推理链</h4>
          <div class="reasoning-chain">
            <span class="chain-node">{前提A}</span><span class="chain-arrow">→</span>
```

### Tab 3: 知识脑图 (Knowledge Mind Map)

**核心区域：SVG 知识脑图，展示书籍完整逻辑结构**

此 Tab 不再使用树形目录，而是用 SVG 渲染的交互式知识脑图，让读者一眼看出书的逻辑结构。关键知识要点通过颜色、大小和特殊标记突出。

**设计原则：**
- **中心节点** = 书名 + 核心主题（渐变填充，最大尺寸）
- **一级分支** = 各部分/模块（圆角矩形，主色系）
- **二级节点** = 各章节（带边框卡片，含一句话总结）
- **三级节点（叶子）= 关键知识点（小标签）
- **关键概念** 用深色实心标记（🔑 前缀），普通概念用浅色
- **跨域关联** 用橙色虚线连接（如「判断力」贯穿财富与幸福两大部分）
- **底部核心洞察横幅** 总结全书最关键的一句话

```html
<section id="tab-framework" class="tab-panel">
  <div class="section-title">
    <h2>🧠 全书知识脑图</h2>
    <p>一本书的逻辑结构，一目了然</p>
  </div>
  <div class="mindmap-container">
    <svg class="mindmap-svg" viewBox="{根据内容调整}" xmlns="http://www.w3.org/2000/svg">
      <defs>
        <!-- 渐变定义: centerGrad, wealthGrad, happyGrad 等 -->
        <!-- 阴影滤镜: shadowMM, shadowSmMM -->
      </defs>

      <!-- 1. 连接线（从中心到分支到节点到叶子） -->
      <path d="{贝塞尔曲线}" stroke="{颜色}" stroke-width="3" fill="none" opacity="0.35"/>

      <!-- 2. 跨域关联虚线（橙色） -->
      <path d="..." stroke="#FF7D00" stroke-dasharray="6,4" opacity="0.3"/>
      <text>关联说明文字</text>

      <!-- 3. 中心节点 (ellipse) -->
      <g filter="url(#shadowMM)">
        <ellipse cx="{x}" cy="{y}" rx="{rx}" ry="{ry}" fill="url(#centerGrad)"/>
        <text>{书名}</text><text>{副标题}</text>
      </g>

      <!-- 4. 一级分支节点 (rect, 渐变填充) -->
      <g filter="url(#shadowMM)">
        <rect x="..." rx="22" width="..." height="..." fill="url(#branchGrad)"/>
        <text>{部分名}</text>
      </g>

      <!-- 5. 二级章节节点 (rect, 浅色背景+边框) -->
      <g filter="url(#shadowSmMM)">
        <rect x="..." rx="16" width="..." height="..." fill="#E6F0FF" stroke="#0052D9"/>
        <text>① {章节名}</text>
        <text>{一句话总结}</text>
      </g>

      <!-- 6. 三级叶子节点 - 关键概念 (实心深色, 🔑前缀) -->
      <g filter="url(#shadowSmMM)">
        <rect rx="10" fill="#0052D9"/><text>🔑 {关键概念}</text>
      </g>

      <!-- 6b. 三级叶子节点 - 普通概念 (浅色) -->
      <g filter="url(#shadowSmMM)">
        <rect rx="10" fill="#82BFFF"/><text>{重要概念}</text>
      </g>

      <!-- 7. 底部核心洞察横幅 -->
      <g filter="url(#shadowSmMM)">
        <rect rx="16" fill="#FFF7E6" stroke="#FF7D00"/>
        <text>💎 核心洞察</text>
        <text>{全书最关键的一句话}</text>
      </g>

      <!-- 8. 图例 -->
      <g transform="translate(...)">
        <rect/> <!-- 图例容器 -->
        <text>关键概念 / 重要概念 / 章节 / 跨域关联</text>
```

### Tab 3.5: 知识卡片 (Knowledge Cards) ★

**位置：知识脑图之后，章节摘要之前**

此 Tab 是全书最核心的视觉化输出——用多张 SVG 公式图/结构图呈现书中最重要的知识模型。用户通过左右箭头或底部圆点导航切换查看。

#### 设计风格（参考图：公式树形图）

每张 SVG 卡片模仿「纳瓦尔人生系统」参考图的风格：
- **米黄/浅色背景** (`#FDF6EC` / `#F0F4FF` / `#FFF9F0` 等)
- **大标题**（书名/主题 + 彩色关键词）
- **主公式行**（彩色高亮核心概念词）
- **箭头指向**子公式和解释文字
- **颜色编码**：
  - 🔴 红 `#C41E3A` — 核心要素/警示
  - 🔵 蓝 `#0052D9` — 财富/判断力相关
  - 🟢 绿 `#00A86B` — 正向结论/健康
  - 🟠 橙 `#E07B39` — 对比/次要概念
  - 🟣 紫 `#6B3FA0` — 方法论/哲学
- **洞察框**：底部带底色的圆角矩形，含 💡/💎 标题 + 核心总结

#### 建议生成 4-6 张卡片

| # | 卡片主题 | 内容要点 |
|---|---------|---------|
| 1 | 全书核心公式 | 最顶层的人生系统总图（如幸福=健康+财富+人际关系） |
| 2 | 财富创造模型 | 财富=收入+杠杆×回报率 及其分解 |
| 3 | 幸福技能树 | 幸福=现实−期望、欲望管理、活在当下 |
| 4 | 判断力×杠杆 | 三种杠杆对比、判断力的提升路径 |
| 5 | 专长发现法 | 专长定义、发现方法、三大来源 |
| 6 | 运气四象限 | 四种运气的可控性矩阵（可选）|

```javascript
function updateChecklistProgress() {
  const checkboxes = document.querySelectorAll('.checklist-item input[type="checkbox"]');
  const total = checkboxes.length;
  const checked = [...checkboxes].filter(cb => cb.checked).length;
  // Save to localStorage
  const key = 'book-read-' + location.pathname;
  const state = {};
  checkboxes.forEach(cb => { state[cb.dataset.id] = cb.checked; });
  localStorage.setItem(key, JSON.stringify(state));
  // Update UI
  document.getElementById('checklistProgress').style.width = (checked/total*100) + '%';
  document.getElementById('checklistProgressText').textContent = `${checked}/${total} 已完成`;
}
```

```javascript
function nextCard() {
```

## Critical Implementation Notes

1. **Self-contained**: ALL CSS and JS must be embedded in the HTML file. No external dependencies.
2. **SVG icons**: Use inline SVG for all icons (arrows, stars, theme toggle). No icon font dependencies.
3. **Radar chart**: Use inline SVG with calculated polygon points. No Chart.js dependency.
4. **Smooth scrolling**: Use `scroll-behavior: smooth` and IntersectionObserver for scroll reveals.
5. **Data embedding**: All book content is embedded as JavaScript objects within a `<script>` tag.
6. **Performance**: Use CSS transforms for animations (GPU-accelerated). Avoid layout thrashing.
7. **SEO**: Include proper `<title>`, `<meta>`, and heading hierarchy even though it's a standalone file.
